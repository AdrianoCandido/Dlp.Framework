Setting up the DLP Repository

Before using the DLP SDK, you have to configure your Visual Studio IDE to access the DLP Repository, witch is available via NuGet. The following steps illustrates how to setup your environment.

Configuring Visual Studio

Visual Studio 2013 is used to illustrate the configuration of DLP Repository, but the example also works with Visual Studio 2012 version.

Adding the DLP Nuget Repository

  1. Open the Package Manager Settings, through menu TOOLS > NuGet Package Manager > Package Manager Settings.

  2. Select Package Sources in the left pane.

  3. Click the Add button, on the right side, to create a new source.

  4. Give the name DLP Repository to the package source.

  5. Set the source address to http://dlpnugetrepository.azurewebsites.net/nuget

  6. Click the Update button, to set the new DLP Repository.

  7. Click OK to finish.

Verifying the DLP Repository

To verify if the DLP Repository was configured correctly, you have to open an existing solution or create a new one.

Validating the DLP Repository

  1. In Solution Explorer, right click the solution name and choose Manage NuGet Packages for Solution...

  2. In the Manage NuGet Packages dialog, check if the DLP Repository entry was added under the Online node.

If the DLP entry is present, than the configuration was done correctly. Otherwise, review the Adding the DLP NuGet Repository section for any missing or mistake step.
